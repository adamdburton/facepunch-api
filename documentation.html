Request Format

All GET parameters to actions are appended to the url as values. For example, the /auth/login action requires username and password, which would be formatted . POST parameters are passed as normal.

For requests to actions in modules requiring authentication, append the session_id returned from one of the /auth actions to the end of the url.

Response Format

All responses are in json format.

Successful requests will return a response with a property of status set to success and a success property containing the response data.

Failed or erroneous request will return a response with a property of status set to error and an error property containing the error.
